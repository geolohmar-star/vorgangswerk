Teil 1 – Formular-Editor

1. Überblick und Einstieg

Der Formular-Editor ist ein visueller Graph-Editor – ähnlich einem Flussdiagramm. Jeder Kasten (Schritt) entspricht einer Seite im Formular, jede Linie (Transition) zeigt den Weg zum nächsten Schritt.

Einstieg: Navigieren Sie zu Formulare → Formular-Editor und klicken Sie + Neues Formular. Vergeben Sie einen Namen und ein Kürzel (3–5 Großbuchstaben).

Der Editor zeigt die Schritte als Kästen im Canvas. Sie können Kästen verschieben, vergrößern und verbinden. Oben links finden Sie die Werkzeuge: Speichern, Schritt hinzufügen, Verbinden, Vorschau.

2. Schritte anlegen

Ein Schritt ist eine Seite im Formular – der Antragsteller sieht alle Felder eines Schritts auf einmal.

  • Klicken Sie im Editor auf + Schritt – ein neuer Kasten erscheint im Canvas
  • Doppelklicken Sie den Kasten, um ihn zu benennen
  • Der erste Schritt muss als START markiert sein
  • Der letzte Schritt muss als ENDE markiert sein – hier erscheint die Zusammenfassung
Empfehlung: Thematisch zusammengehörige Felder in einem Schritt bündeln (z.B. alle Adressfelder in einem Schritt „Neue Adresse"). Nicht mehr als 8–10 Felder pro Schritt.

3. Felder hinzufügen

Klicken Sie auf einen Schritt → rechts öffnet sich das Feldformular. Klicken Sie + Feld hinzufügen und wählen Sie den Feldtyp.

Pflichtfelder: Aktivieren Sie die Option Pflichtfeld, damit das Formular erst weitergeht wenn das Feld ausgefüllt ist. Pflichtfelder werden mit einem roten Sternchen markiert.

4. Übergänge (Transitionen)

Transitionen verbinden zwei Schritte. Ohne Transition kommt das Formular nicht weiter.

  • Klicken Sie im Editor auf den Verbinden-Button und ziehen Sie vom Quell- zum Zielschritt
  • Eine einfache Transition ohne Bedingung führt immer zum nächsten Schritt
  • Mehrere Transitionen vom selben Schritt ermöglichen Verzweigungen
Reihenfolge: Wenn mehrere Transitionen vom selben Schritt ausgehen, bestimmt die Reihenfolge (0, 1, 2 …) die Auswertungsreihenfolge. Die erste Transition deren Bedingung erfüllt ist wird genommen.

5. Bedingte Verzweigungen – Wenn/Dann

Bedingte Transitionen führen den Antragsteller unterschiedliche Wege, abhängig von seiner Eingabe.

Syntax: feld_id == 'Wert'
Beispiele:
  • antragstyp == 'Neubau'
  • weitere_person == 'ja'
  • anzahl > 5
  • geschlecht == 'weiblich'

Aufbau einer Verzweigung:

  1. Radio-Feld anlegen mit den Optionen (z.B. „An-" / „Abmeldung")
  2. Zwei Transitionen vom selben Schritt anlegen
  3. Erste Transition: Bedingung typ == 'Anmeldung', Reihenfolge 0
  4. Zweite Transition: Bedingung typ == 'Abmeldung', Reihenfolge 1
  5. Beide Wege können am Ende in einem gemeinsamen Abschlussschritt zusammenlaufen
Achtung: Der Wert in der Bedingung muss exakt mit der Option im Radio-Feld übereinstimmen – Groß-/Kleinschreibung beachten. 'ja' ≠ 'Ja'.

6. Wiederholungen (Loops)

Ein Loop ermöglicht es, denselben Schritt mehrfach zu durchlaufen – z.B. beliebig viele Personen oder Bäume erfassen.

Wann brauche ich einen Loop? Wenn der Antragsteller eine unbekannte Anzahl gleichartiger Einträge machen soll (z.B. „Person 2, Person 3 …"). Für eine feste Anzahl reichen mehrere Schritte.

Loop-Aufbau – 4 Schritte sind nötig:

Gate
„Weitere Person?"
Radio: ja / nein
→ ja
Loop-Trigger
Zähler-Schritt
Systemfeld: loop_zaehler
→
Loop-Body
Person (weitere)
Felder eintragen
→
Gate (zurück)
„Weitere Person?"
⟲ LOOP
Wie funktioniert der Loop-Mechanismus?
Beim ersten Durchlauf besucht das System Gate → Trigger → Body in dieser Reihenfolge. Sobald der Antragsteller den Loop-Body abschickt, navigiert das System automatisch zurück zum Gate-Schritt. Da der Gate-Schritt bereits besucht wurde, erkennt das System den Loop und archiviert die Eingaben dieser Iteration sofort – bevor der Antragsteller Gate erneut sieht. Der Antragsteller wählt dann „ja" (weiterer Eintrag) oder „nein" (fertig).

Checkliste: Loop korrekt aufbauen

  • Gate-Schritt hat ein Radio-Feld (ja/nein) mit aktiviertem pdf_ausblenden – das Gate-Radio erscheint nicht in der PDF-Zusammenfassung
  • Loop-Trigger-Schritt hat genau ein Systemfeld vom Typ loop_zaehler und das Attribut loop_bezeichnung gesetzt (z.B. „Person")
  • Transition Gate → Loop-Trigger: Bedingung weitere_person == 'ja', Reihenfolge 0
  • Transition Loop-Trigger → Loop-Body: keine Bedingung
  • Transition Loop-Body → Gate: keine Bedingung – diese Transition löst den Loop-Archivierungsmechanismus aus
  • Transition Gate → Nächster Schritt: Bedingung weitere_person == 'nein', Reihenfolge 1
Häufiger Fehler: Die Transition Loop-Body → Gate darf nicht bedingt sein und darf nicht direkt zum Loop-Trigger springen. Nur über den Gate-Schritt funktioniert der Archivierungsmechanismus korrekt – weil der Gate-Schritt als bereits besuchter Schritt den Loop auslöst.
Mehrere Loops: Ein Formular kann mehrere Loops haben (z.B. Wohnungen UND Personen). Jeder Loop braucht eigene Gate-, Trigger- und Body-Schritte. Feld-IDs müssen unterschiedlich sein (z.B. wohnung_zaehler vs. personen_zaehler). Loops dürfen sich nicht verschachteln.

7. Gemeindekennzahl-Feld & PLZ-Suche

Das Feld vom Typ gemeindekennzahl ist ein Spezialfeld für den Amtlichen Gemeindeschlüssel (AGS).

  • Der Antragsteller kann den 8-stelligen AGS direkt eingeben oder über die PLZ-Suche finden
  • Nach Eingabe einer gültigen Postleitzahl schlägt das System passende Gemeinden vor (via OpenPLZ-API)
  • Bei eindeutigem Treffer wird der AGS automatisch eingetragen
  • Das Feld befüllt automatisch weitere Felder: Gemeinde, Kreis, Bundesland (via Autofill)
Feldnamen-Konvention: Trägt das GKZ-Feld die ID neue_gkz, dann werden die Autofill-Quellen automatisch als neue_gkz_gemeinde, neue_gkz_kreis, neue_gkz_land bereitgestellt.

8. Autofill – Felder automatisch befüllen

Felder vom Typ autofill werden automatisch aus einer anderen Quelle befüllt. Der Antragsteller kann den Wert jederzeit überschreiben.

  • Feld anlegen, Typ autofill wählen
  • Im Attribut quelle den Quellnamen angeben, z.B. neue_gkz_gemeinde
  • Das Feld erscheint vorausgefüllt sobald die Quelle einen Wert liefert

9. Formular testen

Klicken Sie im Editor auf Vorschau oder navigieren Sie zur öffentlichen Formular-URL. Prüfen Sie:

  • Alle Pflichtfelder funktionieren
  • Bedingte Übergänge führen zu den richtigen Schritten
  • Bei Loops: mehrere Iterationen anlegen und die Zusammenfassung kontrollieren
  • Die automatisch erzeugte PDF sieht vollständig aus

10. Export & Import

Formulare können als JSON-Datei exportiert und auf anderen Instanzen importiert werden.

  • Export: Im Editor auf Exportieren klicken – eine .json-Datei wird heruntergeladen
  • Import: Unter Formulare → Importieren die JSON-Datei hochladen
  • Alle Schritte, Felder und Transitionen werden übertragen

Teil 2 – OnlyOffice-Briefvorlage

11. Wie funktioniert die automatische Rückantwort?

Vorgangswerk kann nach dem Abschluss eines Formulars automatisch eine personalisierte E-Mail mit PDF-Anhang versenden. Dafür brauchen Sie:

  1. Eine OnlyOffice-Briefvorlage (.docx) mit Platzhaltern
  2. Einen Aktions-Schritt im Formular-Editor der die Vorlage auslöst
Das System befüllt die Platzhalter mit den Antragsdaten, wandelt das Dokument in ein PDF um und sendet es per E-Mail – ohne Zutun des Sachbearbeiters.

12. Vorlage in OnlyOffice erstellen

  1. Öffnen Sie OnlyOffice (Menü: Dokumente) und erstellen Sie ein neues Textdokument
  2. Gestalten Sie die Vorlage wie einen normalen Brief – Briefkopf, Anrede, Text, Grußformel
  3. An den Stellen, die befüllt werden sollen, fügen Sie Platzhalter ein (siehe Abschnitt 13)
  4. Speichern Sie das Dokument als .docx

13. Platzhalter einfügen

Platzhalter haben das Format {{feld_id}}. Beim Befüllen wird der Platzhalter durch den tatsächlichen Wert ersetzt.

PlatzhalterInhalt
{{empfaenger_name}}Name des Antragstellers
{{empfaenger_strasse}}Straße des Antragstellers
{{empfaenger_plz_ort}}PLZ und Ort
{{datum}}Heutiges Datum
{{betreff}}Betreff des Schreibens
{{anrede}}Anrede (z.B. „Sehr geehrte Damen und Herren")
{{brieftext}}Individueller Brieftext
{{vorgangsnummer}}Automatisch vergebene Vorgangsnummer
{{feld_id}}Beliebiges Formularfeld (ID aus dem Editor)
{{loop_0_feld_id}}Feld aus dem 1. Loop-Durchlauf
{{loop_1_feld_id}}Feld aus dem 2. Loop-Durchlauf
Wichtig: Der Platzhalter-Name muss exakt mit der Feld-ID im Editor übereinstimmen – Groß-/Kleinschreibung, Unterstriche, keine Leerzeichen.

Die vollständige Platzhalter-Liste für Ihre Vorlage finden Sie unter Korrespondenz → Vorlagen → Platzhalter anzeigen.

14. Vorlage hochladen & verknüpfen

  1. Navigieren Sie zu Korrespondenz → Vorlagen
  2. Klicken Sie + Neue Vorlage
  3. Geben Sie einen Namen ein, z.B. „Bestätigung Ummeldung Hauptwohnsitz"
  4. Laden Sie die .docx-Datei hoch und wählen Sie ggf. das zugehörige Formular (Kürzel)
  5. Speichern Sie – die Vorlage erhält eine interne Vorlagen-ID (Zahl in der URL)
  6. Notieren Sie diese ID – Sie benötigen sie im nächsten Schritt

15. Aktions-Schritt im Formular einrichten

Der Aktions-Schritt ist ein spezieller Schritt im Formular-Editor, der nach dem Abschluss automatisch ausgeführt wird – ohne dass der Antragsteller etwas tun muss.

  1. Fügen Sie im Editor einen neuen Schritt nach dem Ende-Schritt ein
  2. Setzen Sie den Typ auf Aktions-Schritt
  3. Wählen Sie die Aktion: E-Mail mit Vorlage senden
  4. Tragen Sie die Vorlagen-ID aus Schritt 14 ein
  5. Tragen Sie im Empfänger-Feld die Feld-ID des E-Mail-Felds ein, z.B. {{email}}
  6. Speichern und testen
Fertig! Nach dem Speichern sendet Vorgangswerk bei jedem Abschluss dieses Formulars automatisch eine personalisierte E-Mail mit der ausgefüllten Vorlage als PDF-Anhang – ohne weiteres Zutun des Sachbearbeiters.

16. Formular per KI aus PDF importieren (Portal)

Im Vorgangswerk-Portal kann ein bestehendes PDF-Formular hochgeladen werden. Claude analysiert das Dokument und erstellt automatisch einen fertigen Antrags-Pfad – inklusive Schritte, Felder, Typen und Transitionen.

Farbmarker-Konvention

Bereite das PDF mit farbigen Markierungen vor (z.B. in Word, LibreOffice oder einem PDF-Editor). Die KI erkennt die Farben als Strukturierungsanweisungen:

Farbe Bedeutung Beispiel-Text
Blau Loop – wiederholbarer Abschnitt LOOP: Kind
Grün Schritt/Gruppe – neuer Schritt GRUPPE: Wohnort
Gelb Split – Feld aufteilen SPLIT: PLZ, Gemeinde
Türkis Autofill – automatisch befüllen AUTOFILL: neue_gkz_gemeinde
Violett Zeige wenn – bedingtes Feld ZEIGE_WENN: hat_hund = ja
Rot (Zahl) Verzweigung – Entscheidungsweg 3 / 4 / 5
Rot (voll) Ignorieren – Felder weglassen X (oder kein Text)

Bedingte Felder (Zeige-wenn)

Typisches Muster: Radio-Feld „Haben Sie einen Hund?" (ja/nein) – das Folgefeld „Name des Hundes" erscheint nur bei „ja".

  1. Radio-Feld hat_hund mit Optionen ja / nein anlegen
  2. Folgefeld mit violettem Marker: ZEIGE_WENN: hat_hund = ja
  3. Im Editor: Feld-Attribut zeige_wenn auf hat_hund:ja setzen

Die KI erkennt dieses Muster auch ohne Marker automatisch, wenn nach einem Ja/Nein-Feld ein Feld mit „Falls ja …" folgt.

Ablauf im Portal: PDF hochladen → KI analysiert (20–60 Sek.) → Prüfschritt: Feldtypen & Optionen anpassen → Importieren → Pfad im Editor nachbearbeiten.
