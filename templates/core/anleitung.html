Teil 1 – Formular-Editor

{# 1 Überblick #}

1. Überblick und Einstieg

Der Formular-Editor ist ein visueller Graph-Editor – ähnlich einem Flussdiagramm. Jeder Kasten (Schritt) entspricht einer Seite im Formular, jede Linie (Transition) zeigt den Weg zum nächsten Schritt.

🗐 Schritte können per Maus verschoben werden · Doppelklick öffnet die Bearbeitung
So öffnen Sie den Editor: Menü → Formulare → Formular in der Liste auswählen → Button „Bearbeiten". Der Editor öffnet sich als Vollbild-Graph im Browser.
{# 2 Schritte anlegen #}

2. Schritte anlegen

Jeder Schritt entspricht einer Seite, die der Antragsteller ausfüllt. Planen Sie zunächst auf Papier welche Themen logisch zusammengehören.

📷 Schritt anlegen – Werkzeugleiste oben rechts
+ Schritt ← Klick auf diesen Button fügt einen neuen Schritt in der Mitte des Editors ein
Schritt-Dialog (öffnet sich per Doppelklick auf den Schritt):
Schritt bearbeiten
Wichtig: Jedes Formular braucht genau einen Startschritt (Haken bei „Ist Startschritt") und mindestens einen Endschritt (Haken bei „Ist Endschritt"). Beim Abschluss des Endschritts wird der Antrag eingereicht.
{# 3 Felder #}

3. Felder hinzufügen

Innerhalb eines Schritts fügen Sie Felder hinzu – das sind die eigentlichen Eingabeelemente des Formulars. Klicken Sie doppelt auf einen Schritt → Reiter „Felder" → „+ Feld".

{# 4 Transitionen #}

4. Übergänge (Transitionen)

Transitionen verbinden Schritte miteinander. Ohne Transition kommt der Antragsteller nicht weiter.

📷 Transition ziehen – so geht es
Schritt A
Person 1
⟶
Linie ziehen
Schritt B
Weitere Person?
So ziehen Sie eine Transition: Bewegen Sie die Maus an den Rand eines Schritts bis ein kleiner Kreis (⚬) erscheint → halten Sie die Maustaste gedrückt → ziehen Sie zur auf den Ziel-Schritt → loslassen. Die Verbindungslinie erscheint automatisch.
{# 5 Bedingte Verzweigungen #}

5. Bedingte Verzweigungen – Wenn/Dann

Mit Bedingungen steuern Sie welchen Weg der Antragsteller nimmt – abhängig von seinen Angaben. Klicken Sie auf eine Transition (Linie) um die Bedingung einzutragen.

{# 6 Loops #}

6. Wiederholungen (Loops)

Ein Loop ermöglicht es, denselben Schritt mehrfach zu durchlaufen – z.B. beliebig viele Personen oder Wohnungen anzumelden.

Wann brauche ich einen Loop? Wenn der Antragsteller eine unbekannte Anzahl gleichartiger Einträge machen soll (z.B. „Person 2, Person 3, Person 4 …"). Für eine feste Anzahl reichen mehrere Schritte.
📷 Loop-Aufbau – 4 Schritte sind nötig
Gate
Weitere Person?
Radio: ja/nein
⟶ ja
Loop-Trigger
Zähler-Schritt
Systemfeld: loop_zaehler
⟶
Loop-Body
Person (weitere)
Felder eintragen
⟶
Gate (zurück)
Weitere Person?
↶ LOOP
Funktionsweise: Der Loop-Trigger-Schritt wird beim ersten Durchlauf normal besucht. Wenn der Antragsteller „ja" wählt und der Loop-Body ausgefüllt hat, kehrt er zum Gate-Schritt zurück. Wählt er dort erneut „ja" → der Loop-Trigger wurde bereits besucht → das System erkennt den Loop und archiviert die Daten automatisch als Iteration. Bei „nein" geht es normal weiter.
✅ Checkliste: Loop korrekt aufbauen
  • Gate-Schritt hat ein Radio-Feld (ja/nein) mit pdf_ausblenden: ✓
  • Loop-Trigger-Schritt hat genau ein Systemfeld vom Typ loop_zaehler
  • Transition Gate → Loop-Trigger: Bedingung weitere_person == 'ja', Reihenfolge 0
  • Transition Loop-Trigger → Loop-Body: keine Bedingung
  • Transition Loop-Body → Gate: keine Bedingung (erzeugt den Loop-Rücksprung)
  • Transition Gate → Nächster Schritt: Bedingung weitere_person == 'nein', Reihenfolge 1
Mehrere unabhängige Loops: Ein Formular kann mehrere Loops haben (z.B. Loop für Wohnungen UND Loop für Personen). Jeder Loop braucht seinen eigenen Gate-Schritt, Loop-Trigger und Loop-Body. Die Feld-IDs müssen unterschiedlich sein (z.B. wohnung_zaehler vs. personen_zaehler).
{# 7 Gemeindekennzahl #}

7. Gemeindekennzahl-Feld & PLZ-Suche

Das Feld vom Typ gemeindekennzahl ist ein Spezialfeld für den Amtlichen Gemeindeschlüssel (AGS). Es enthält eine automatische PLZ-Suche – der Antragsteller tippt einfach seine Postleitzahl, das System schlägt die passende Gemeinde vor.

📷 Gemeindekennzahl-Feld im Formular (Ansicht des Antragstellers)
Gemeindekennzahl (neue Wohnung) *
PLZ eingeben → Gemeinde wird gesucht
8-stellig, wird automatisch gefüllt

Die GKZ-Unterfelder (neue_gkz_gemeinde, neue_gkz_kreis, neue_gkz_land) werden automatisch mitgespeichert und können als Autofill-Quelle in anderen Feldern genutzt werden.

{# 8 Autofill #}

8. Autofill – Felder automatisch befüllen

Ein Feld vom Typ autofill wird automatisch mit dem Wert eines anderen Feldes befüllt. Das ist nützlich wenn dieselbe Information an mehreren Stellen erscheinen soll (z.B. Gemeinde aus der Gemeindekennzahl in ein separates Textfeld übertragen).

📷 Autofill einrichten – Feldtyp = autofill
Feld bearbeiten
Wählen Sie das Quellfeld. GKZ-Felder bieten _gemeinde, _kreis, _land, _plz als Unterfelder.
{# 9 Testen #}

9. Formular testen

Bevor Sie ein Formular freigeben, testen Sie den vollständigen Ausfüllprozess.

📷 Test-Schaltfläche im Editor
▶ Formular testen ← Öffnet das Formular in einem neuen Tab als Testlauf (wird nicht als echter Antrag gezählt)
  • Prüfen Sie alle Pfade: „ja"- und „nein"-Zweige durchklicken
  • Prüfen Sie Pflichtfelder: leere Pflichtfelder müssen einen Fehler zeigen
  • Prüfen Sie das PDF: nach Abschluss erscheint ein Download-Link für das PDF
  • Prüfen Sie Loops: mehrere Iterationen anlegen und die Zusammenfassung kontrollieren

Teil 2 – Automatische Rückantwort mit OnlyOffice-Briefvorlage

{# 11 Wie funktioniert es #}

11. Wie funktioniert die automatische Rückantwort?

Nach dem Einreichen eines Antrags kann Vorgangswerk automatisch eine personalisierte Bestätigung per E-Mail versenden – mit einem ausgefüllten PDF-Anhang aus einer Word-Vorlage (.docx).

📷 Ablauf der automatischen Rückantwort
1
Antragsteller schließt das Formular ab
|
2
System erkennt den Aktions-Schritt im Formular
|
3
Antragsdaten werden in die DOCX-Vorlage eingefügt (Platzhalter ersetzt)
|
4
DOCX wird in PDF umgewandelt (OnlyOffice oder Fallback)
|
5
E-Mail mit PDF-Anhang geht an den Antragsteller und/oder die Behörde

{# 13 Platzhalter #}

13. Platzhalter einfügen

Platzhalter werden beim Versand automatisch durch die Antragsdaten ersetzt. Die Syntax ist: doppelte geschweifte Klammern um die Feld-ID.

📷 Platzhalter-Syntax
Platzhalter in der VorlageWird ersetzt durchBeispielwert
{% verbatim %}{{p1_familienname}}{% endverbatim %} Familienname Person 1 Müller
{% verbatim %}{{neue_plz}}{% endverbatim %} PLZ neue Wohnung 80331
{% verbatim %}{{datum_aenderung}}{% endverbatim %} Datum der Änderung 2026-04-16
{% verbatim %}{{neue_gkz_gemeinde}}{% endverbatim %} Gemeindename (aus GKZ-Feld) München
Schreibweise beachten: Die Feld-ID im Platzhalter muss exakt mit der Feld-ID im Editor übereinstimmen – Groß-/Kleinschreibung, Unterstriche, keine Leerzeichen.
{# 14 Vorlage hochladen #}

14. Vorlage hochladen & verknüpfen

Die fertige .docx-Vorlage laden Sie als Briefvorlage in Vorgangswerk hoch.

📷 Vorlage anlegen – Menü Korrespondenz → Vorlagen
  1. Navigieren Sie zu Korrespondenz → Vorlagen
  2. Klicken Sie + Neue Vorlage
  3. Geben Sie einen Namen ein z.B. „Bestätigung Ummeldung Hauptwohnsitz"
  4. Laden Sie die .docx-Datei hoch
  5. Speichern Sie – die Vorlage erhält eine interne Vorlagen-ID (Zahl)
  6. Notieren Sie diese ID – Sie benötigen sie im nächsten Schritt
{# 15 Aktions-Schritt #}

15. Aktions-Schritt im Formular einrichten

Der Aktions-Schritt ist ein spezieller Schritt im Formular-Editor, der nach dem Abschluss automatisch ausgeführt wird – ohne dass der Antragsteller etwas tun muss.

📷 Aktions-Schritt anlegen
  1. Fügen Sie im Editor einen neuen Schritt ein: nach dem Endschritt
  2. Doppelklick auf den Schritt → aktivieren Sie „Ist Aktions-Schritt"
  3. Fügen Sie ein Feld vom Typ aktion_briefvorlage hinzu
  4. Tragen Sie die Vorlagen-ID aus Schritt 14 ein
  5. Tragen Sie die Empfänger-E-Mail ein oder verweisen Sie auf ein Formularfeld
⚡ Aktions-Schritt – Feldkonfiguration
ID der Briefvorlage aus dem Vorlagen-Manager
Falls das Formular ein E-Mail-Feld enthält, schickt das System die Bestätigung direkt an den Antragsteller
✓ Fertig! Nach dem Speichern sendet Vorgangswerk bei jedem Abschluss dieses Formulars automatisch eine personalisierte E-Mail mit der ausgefüllten Vorlage als PDF-Anhang – ohne weiteres Zutun des Sachbearbeiters.
