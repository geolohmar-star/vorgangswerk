Root-CA Importanleitung

Windows Gruppenrichtlinie (GPO)
Fuer: IT-Administratoren  |  System: PRIMA Intranet  |  Zweck: Vertrauenswuerdige interne Zertifizierungsstelle einrichten

1. Warum ist dieser Schritt noetig?

PRIMA verwendet eine interne Root-Zertifizierungsstelle (Root-CA) um digitale Signaturen auf PDFs zu erzeugen. Mitarbeiter-Zertifikate werden automatisch beim Anlegen eines HR-Profils ausgestellt.

Ohne diesen Einrichtungsschritt zeigen Adobe Acrobat und andere PDF-Viewer die Signatur als "Signatur unbekannt" oder "Zertifikat nicht vertrauenswuerdig" an, weil die Root-CA nicht im Windows-Zertifikatspeicher bekannt ist.

Ziel: Nach dieser Anleitung zeigt Adobe Acrobat alle PRIMA-signierten PDFs mit "Alle Signaturen sind gueltig" (gruenes Haekchen).

2. Root-CA-Zertifikat herunterladen

1Als Administrator in PRIMA einloggen
Navigation: Signatur → Dashboard
2Zertifikat herunterladen
Link: Signatur → CA-Zertifikat herunterladen
Die Datei ca_root.cer wird heruntergeladen (DER-Format, direkt fuer Windows geeignet).

3. Gruppenrichtlinie einrichten (GPO)

Voraussetzung: Active Directory-Domaeene vorhanden. Ohne AD: Schritt 4 (manueller Import) verwenden.
1Gruppenrichtlinienverwaltung oeffnen
Win + R → gpmc.msc → Enter
2Neue GPO erstellen oder vorhandene bearbeiten
Rechtsklick auf die gewuenschte OU oder die Domaene → "Gruppenrichtlinienobjekt hier erstellen und verknuepfen"
Name: PRIMA – Interne Root-CA verteilen
3GPO bearbeiten
Pfad im GPO-Editor:
Computerkonfiguration → Windows-Einstellungen → Sicherheitseinstellungen → Richtlinien oeffentlicher Schluessel → Vertrauenswuerdige Stammzertifizierungsstellen
4Zertifikat importieren
Rechtsklick im rechten Fensterbereich → Importieren
Datei auswaehlen: ca_root.cer
Speicher bestaetigen: "Alle Zertifikate in folgendem Speicher speichern" → Vertrauenswuerdige Stammzertifizierungsstellen
5GPO speichern und verknuepfen
GPO-Editor schliessen. In der Gruppenrichtlinienverwaltung pruefen: GPO muss mit der richtigen OU oder Domaene verknuepft sein.
6Richtlinie anwenden
Auf einem Testrechner ausfuehren:
gpupdate /force
Oder warten auf naechste Anmeldung (GPO wird automatisch alle 90 Minuten aktualisiert).

4. Manueller Import (ohne Active Directory)

Fuer einzelne PCs oder ohne AD-Infrastruktur:

1Zertifikat importieren
Doppelklick auf ca_root.cer → "Zertifikat installieren" → "Lokaler Computer" (nicht aktueller Benutzer!) → "Alle Zertifikate in folgendem Speicher speichern" → Vertrauenswuerdige Stammzertifizierungsstellen → Fertig stellen
Administratorrechte erforderlich! "Lokaler Computer" gilt fuer alle Benutzer des PCs, "Aktueller Benutzer" nur fuer den angemeldeten Account.

5. Erfolg pruefen

Tool Pruefen Erwartetes Ergebnis
certmgr.msc Vertrauenswuerdige Stammzertifizierungsstellen → Zertifikate "Interne Root CA" oder "PRIMA Root CA" muss sichtbar sein
Adobe Acrobat PRIMA-PDF oeffnen → Signatur-Panel Gruenes Haekchen: "Alle Signaturen sind gueltig"
Chrome / Edge Nutzen Windows-Zertifikatspeicher automatisch Kein separater Schritt noetig nach GPO
Firefox about:config → security.enterprise_roots.enabled Auf true setzen, damit Firefox den Windows-Store nutzt
Verifikation in PRIMA: Unter Signatur → PDF pruefen kann jede signierte PDF-Datei direkt im Browser validiert werden – unabhaengig vom lokalen Zertifikatspeicher.

6. Haeufige Probleme

Symptom Loesung
Adobe zeigt nach GPO immer noch "unbekannt" Adobe benoetigt Neustart nach Zertifikatsaenderung. Evtl. auch gpupdate /force wiederholen.
certmgr.msc zeigt kein PRIMA-Zertifikat GPO-Verknuepfung pruefen. Sicherstellen dass "Lokaler Computer" (nicht Benutzer) gewaehlt wurde.
Firefox zeigt trotz GPO Warnung about:config → security.enterprise_roots.enabled = true setzen.
Signatur zeigt falschen Namen Zertifikat des Mitarbeiters pruefen: Signatur-Dashboard → Zertifikat-Liste.
